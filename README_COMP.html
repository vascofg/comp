SUMMARY:

Our tool reads a regular expression, converts it to an automaton, and then turns it into java code.

DEALING WITH SYNTACTIC ERRORS:

The program, when dealing with syntactic errors, alerts the user about the error that occurred and gives him the opportunity to make a new insertion. The program closes only when requested by the user.

SEMANTIC ANALYSIS:

Semantic analysis does not cover our work since we're working with regular expressions.

INTERMEDIATE REPRESENTATIONS (IRs):

At this work we only use abstract syntactic tree (AST), which is an high-level intermediate representation of the introduced regular expression after syntactically analyzed.

CODE GENERATION:

Our tool has some class files such as Automaton, State and Connection which are essential to represent the generated automata. The automata are stored as a Serializable class which is loaded when the user runs the generated java code.

OVERVIEW:

We use an algorithm developed by us to convert regular expressions to DFA.

TESTSUITE AND TEST INFRASTRUCTURE:

Our test suite has 6 automata generated from regular expressions with the help of the site HackingOff and compares them with those generated by our program.

HOW TO USE:

We have two jars included in the ZIP which can be used to generate and implement the automata.
Generate.jar takes a regex and converts it to an automaton, saving it in the same folder with a chosen name. It also creates a DOT file and an SVG image if Graphviz is installed.
Implement.jar reads this file and implements the automaton: it takes an input string and verifies if it passes in the automaton or not.

CONS:
